Predict Server Failures with Azure AI

The Server Failure Predictor leverages Azure OpenAI's gpt-4o-mini model to analyze server metrics and predict the likelihood of server failure within the next 24 hours. It processes metrics such as CPU usage, memory usage, disk I/O, network I/O, and storage usage, generating predictions with confidence scores and detailed explanations.

Key Features

  • Input: server_data.csv with columns: timestamp, host, cpu_usage_percent, memory_usage_percent, disk_io_mb_s, network_io_mb_s, storage_used_gb.
  • Output: predictions.csv with columns: host, prediction, confidence, explanation; also displayed on the console.
  • Prerequisites: Python 3.7+, openai, pandas, numpy; Azure OpenAI resource with gpt-4o-mini deployed.
  • Configuration: Requires azure_foundry.txt with endpoint, key, model, and deployment details.
  • Usage: python3 scripts/server_metrics_failure_predictor_azure_ai.py server_data.csv
